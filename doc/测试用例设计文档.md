# 网站测绘 CLI 工具 MVP 测试用例设计文档

## 1. 测试目标

本文档基于《架构与MVP任务文档.md》设计测试用例，用于校验网站测绘 CLI 工具 MVP 各模块和完整链路是否正确。

核心验证目标：

- CLI 参数能被正确解析和校验。
- Mock 数据能稳定驱动完整流程。
- mDNS 原始记录能被解析为标准化中间结构。
- PTR/SRV/TXT/A/AAAA 记录能聚合为服务资产。
- CIDR 和端口范围过滤符合需求规则。
- text/json 输出结构稳定、字段完整、banner 深度满足示例要求。
- 真实 mDNS 查询在有设备的局域网中能返回可解析记录。
- 错误输入和异常数据能被明确处理，不导致程序崩溃。

## 2. 测试范围

### 2.1 范围内

- `cmd/mdnsmap` CLI 主流程。
- `internal/config` 参数解析与校验。
- `internal/mdns` mock loader、真实 mDNS 查询适配层。
- `internal/parser` RawRecord 到 ParsedRecord 解析。
- `internal/asset` 资产聚合。
- `internal/filter` IP 与端口过滤。
- `internal/output` text/json 输出。
- `testdata/sample-mdns.json` 示例数据。

### 2.2 范围外

- TCP 主动端口扫描。
- HTTP title/banner 二次探测。
- SMB/AFP 协议登录或深度探测。
- 漏洞验证。
- 分布式扫描。
- Web UI。
- 数据库存储。

## 3. 测试分层

| 层级 | 目标 | 推荐实现方式 |
|---|---|---|
| 单元测试 | 验证单模块纯逻辑 | Go `testing` 包 |
| 集成测试 | 验证 mock 数据主链路 | Go `testing` 包或 CLI 命令 |
| CLI 验收测试 | 验证用户命令入口、输出、退出码 | `go run ./cmd/mdnsmap ...` |
| 真实环境测试 | 验证真实 mDNS 网络能力 | 局域网手工/半自动测试 |

## 4. 标准 Mock 数据集

建议使用 `testdata/sample-mdns.json` 作为主验收数据集，至少包含以下资产记录。

### 4.1 NAS 正常记录

包含同一主机 `slw-nas.local` 的多服务：

- `_workstation._tcp.local`，端口 `9`。
- `_http._tcp.local`，端口 `5000`，TXT 包含 `path=/`。
- `_smb._tcp.local`，端口 `445`。
- `_qdiscover._tcp.local`，端口 `5000`，TXT 包含：
  - `accessType=https`
  - `accessPort=86`
  - `model=TS-X64`
  - `displayModel=TS-464C`
  - `fwVer=5.2.9`
  - `fwBuildNum=20260214`
- `_device-info._tcp.local`，无端口或端口为 `0`，TXT 包含 `model=Xserve`。
- `_afpovertcp._tcp.local`，端口 `548`。
- A 记录：`192.168.1.20`。
- AAAA 记录：`fe80::265e:beff:fe69:a313`。

### 4.2 跨网段记录

增加一台主机 `other-nas.local`：

- `_http._tcp.local`，端口 `5000`。
- A 记录：`10.0.0.10`。

用于验证 CIDR 过滤。

### 4.3 端口外记录

增加一条服务：

- 主机 `slw-nas.local`。
- 服务 `_http._tcp.local`。
- 端口 `8080`。

用于验证端口过滤。

### 4.4 IPv6-only 记录

增加一条服务：

- 主机 `ipv6-only.local`。
- 服务 `_http._tcp.local`。
- 端口 `8080`。
- 无 A 记录。
- AAAA 记录：`fe80::1`。

用于验证 `asset.IP` 为空但 `asset.IPv6` 存在时的 MVP 保留规则。

## 5. Config 模块测试用例

### TC-CONFIG-001 合法最小参数

模块：`internal/config`

输入：

```bash
--cidr 192.168.1.0/24 --ports 1-6000
```

期望：

- 解析成功。
- `CIDR=192.168.1.0/24`。
- `Timeout=5s`。
- `Format=text`。
- `Ports` 包含 `1`、`80`、`5000`、`6000`。
- `Ports` 不包含 `6001`。

### TC-CONFIG-002 短参数

输入：

```bash
-i 192.168.1.0/24 -p 80,443 -t 3s -f json
```

期望：

- 解析成功。
- `Format=json`。
- `Timeout=3s`。
- `Ports` 包含 `80`、`443`。

### TC-CONFIG-003 缺少 CIDR

输入：

```bash

```

期望：

- 解析失败。
- 错误信息包含 `cidr is required` 或等价提示。
- CLI 退出码应为非 0，建议为 `2`。

- 解析失败。

### TC-CONFIG-005 非法 CIDR 格式


期望：

## 7. mDNS 模块测试用例

### TC-MDNS-001 Mock 文件读取成功

模块：`internal/mdns`

输入：

```text
testdata/sample-mdns.json
```

期望：

- 返回 `[]RawRecord`。
- 记录数量大于 0。
- 至少包含 `PTR`、`SRV`、`TXT`、`A`、`AAAA`。

### TC-MDNS-002 Mock 文件不存在

输入：

```text
testdata/not-exists.json
```

期望：

- 返回错误。
- 错误信息包含读取文件失败原因。
- CLI 退出码应为非 0，建议为 `1`。

### TC-MDNS-003 Mock JSON 非法

输入文件内容：

```json
{bad json
```

期望：

- 返回 JSON 解析错误。
- 不进入 parser/asset/filter/output 主链路。

### TC-MDNS-004 Mock 字段兼容

输入记录：

```json
[
  {"type":"TXT","name":"slw-nas._http._tcp.local","txt":["path=/"],"ttl":10}
]
```

期望：

- 解析成功。
- 未提供字段保持零值。
- 不因缺少 SRV/A 直接崩溃。

### TC-MDNS-005 默认查询服务类型

模块：`internal/mdns`

期望：

- 默认查询列表包含 `_services._dns-sd._udp.local`。
- 默认查询列表包含 `_http._tcp.local`。
- 默认查询列表包含 `_smb._tcp.local`。
- 默认查询列表包含 `_workstation._tcp.local`。
- 默认查询列表包含 `_device-info._tcp.local`。
- 默认查询列表包含 `_afpovertcp._tcp.local`。
- 默认查询列表包含 `_qdiscover._tcp.local`。

### TC-MDNS-006 真实 mDNS 超时无响应

前置条件：

- 当前网络无 mDNS 设备，或使用极短 timeout。

命令：

```bash
go run ./cmd/mdnsmap --cidr 192.168.1.0/24 --ports 1-6000 --timeout 1s
```

期望：

- 程序按 timeout 返回。
- 输出空 `services:` 或合法空 JSON。
- 退出码为 `0`。
- 不长时间阻塞。

### TC-MDNS-007 真实 mDNS 网络错误

前置条件：

- 模拟 UDP 5353 被系统拒绝、网络接口不可用或无权限。

期望：

- 返回明确错误。
- 错误信息包含权限、网络接口或 UDP 监听发送失败原因。
- CLI 退出码应为非 0，建议为 `1`。

### TC-PARSER-009 PTR 记录解析

输入 RawRecord：

```json

```

- `RecordType=PTR`。
- `Service=http`。
- `Protocol=tcp`。
- `Instance=slw-nas._http._tcp.local` 或等价实例字段。
- `RawName=_http._tcp.local`。
- `RawValue=slw-nas._http._tcp.local`。

### TC-PARSER-010 SRV 记录解析

输入 RawRecord：

```json
{
  "type": "SRV",
  "name": "slw-nas._http._tcp.local",
  "hostname": "slw-nas.local",
  "port": 5000,
  "ttl": 10
}
```

期望 ParsedRecord：

- `RecordType=SRV`。
- `Service=http`。
- `Protocol=tcp`。
- `Name=slw-nas`。
- `Hostname=slw-nas.local`。
- `Port=5000`。
- `TTL=10`。

### TC-PARSER-011 A 记录解析

输入 RawRecord：

```json
{
  "type": "A",
  "name": "slw-nas.local",
  "ipv4": "192.168.1.20",
  "ttl": 10
}
```

期望 ParsedRecord：

- `RecordType=A`。
- `Hostname=slw-nas.local`。
- `IPv4=192.168.1.20`。
- `TTL=10`。

### TC-PARSER-012 AAAA 记录解析

输入 RawRecord：

```json
{
  "type": "AAAA",
  "name": "slw-nas.local",
  "ipv6": "fe80::265e:beff:fe69:a313",
  "ttl": 10
}
```

期望 ParsedRecord：

- `RecordType=AAAA`。
- `Hostname=slw-nas.local`。
- `IPv6=fe80::265e:beff:fe69:a313`。
- `TTL=10`。

### TC-PARSER-013 未知记录类型

输入 RawRecord：

```json
{
  "type": "UNKNOWN",
  "name": "x.local"
}
```

期望：

- 默认跳过或返回可识别错误。
- verbose 模式可输出跳过原因。
- 不影响其他记录解析。

## 9. Asset 聚合模块测试用例

### TC-ASSET-001 聚合同一 HTTP 服务

输入 ParsedRecord：

- PTR：`_http._tcp.local -> slw-nas._http._tcp.local`
- SRV：`slw-nas._http._tcp.local -> slw-nas.local:5000`
- TXT：`slw-nas._http._tcp.local -> path=/`
- A：`slw-nas.local -> 192.168.1.20`
- AAAA：`slw-nas.local -> fe80::265e:beff:fe69:a313`

期望 ServiceAsset：

- 只有一条 HTTP 服务。
- `ip=192.168.1.20`。
- `ipv6=fe80::265e:beff:fe69:a313`。
- `port=5000`。
- `protocol=tcp`。
- `service=http`。
- `name=slw-nas`。
- `hostname=slw-nas.local`。
- `ttl=10`。
- `banner.path=/`。

### TC-ASSET-002 同主机多服务分别输出

输入：

- `slw-nas.local:5000 http`
- `slw-nas.local:445 smb`
- `slw-nas.local:5000 qdiscover`
- `slw-nas.local:548 afpovertcp`

期望：

- 输出 4 条服务资产。
- `http` 和 `qdiscover` 即使端口相同也不能合并为一条。
- 每条服务保留相同 hostname 和 IP。

### TC-ASSET-003 qdiscover 深度 banner 合并

输入：

- SRV：`slw-nas._qdiscover._tcp.local -> slw-nas.local:5000`
- TXT：
  - `accessType=https`
  - `accessPort=86`
  - `model=TS-X64`
  - `displayModel=TS-464C`
  - `fwVer=5.2.9`
  - `fwBuildNum=20260214`

期望：

- 输出一条 `qdiscover` 服务。
- banner 中完整保留 6 个字段。

### TC-ASSET-004 device-info 无端口聚合

输入：

- PTR：`_device-info._tcp.local -> slw-nas(AFP)._device-info._tcp.local`
- TXT：`model=Xserve`
- A/AAAA：`slw-nas.local` 对应 IP。

期望：

- 输出一条 `device-info` 资产。
- `port` 为空或为 `0`。
- `service=device-info`。
- `banner.model=Xserve`。
- 后续过滤阶段应保留该记录。

### TC-ASSET-005 answers PTR 汇总

输入 PTR：

- `_workstation._tcp.local`
- `_http._tcp.local`
- `_smb._tcp.local`
- `_qdiscover._tcp.local`
- `_device-info._tcp.local`
- `_afpovertcp._tcp.local`

期望：

- `ScanResult.Answers.PTR` 包含以上服务类型。
- 重复 PTR 应去重。
- 输出顺序建议稳定，便于测试断言。

### TC-ASSET-006 TTL 合并规则

输入：

- 同一服务不同记录 TTL 分别为 `10`、`120`。

期望：

- 聚合后的 TTL 使用确定性规则。
- 推荐使用最小非零 TTL 或 SRV TTL。
- 测试应固定该规则，避免输出不稳定。

### TC-ASSET-007 缺少 A 记录但有 AAAA

输入：

- SRV/TXT 正常。
- 无 A 记录。
- AAAA：`fe80::1`。

期望：

- 输出资产中 `ip` 为空。
- `ipv6=fe80::1`。
- 不因缺少 IPv4 丢失资产。

### TC-ASSET-008 缺少 SRV 但有 TXT

输入：

- TXT：`slw-nas._http._tcp.local -> path=/`
- A：`slw-nas.local -> 192.168.1.20`

期望：

- 可以跳过该服务或输出不完整资产，但行为必须稳定。
- 不应崩溃。
- verbose 模式建议提示无法关联 hostname/port。

## 13. 真实 mDNS 环境验收用例

### TC-REAL-001 局域网存在 mDNS 设备

前置条件：

- 测试机器与 NAS、打印机、Mac 或其他支持 mDNS 的设备在同一局域网。
- 本机防火墙允许 UDP 5353。

命令：

```bash
go run ./cmd/mdnsmap --cidr 192.168.1.0/24 --ports 1-6000 --timeout 5s --format text
```

期望：

- 程序在 5 秒左右返回。
- 如果网络中有 mDNS 服务，输出至少一条服务资产。
- 输出字段尽量包含 IP、Port、Hostname、Service、Protocol、TTL、Banner。
- 如果设备包含 TXT，banner 应结构化展示。

### TC-REAL-002 JSON 输出真实 mDNS

命令：

```bash
go run ./cmd/mdnsmap --cidr 192.168.1.0/24 --ports 1-6000 --timeout 5s --format json
```

期望：

- 输出合法 JSON。
- 即使未发现服务，也应输出合法空结果。
- 不输出混杂的 debug 文本到 stdout。

### TC-REAL-003 verbose 模式

命令：

```bash
go run ./cmd/mdnsmap --cidr 192.168.1.0/24 --ports 1-6000 --timeout 5s --verbose
```

期望：

- stdout 仍保持正常结果格式。
- 调试信息输出到 stderr。
- 解析失败的单条记录不影响整体输出。

## 14. 回归测试矩阵

| 用例组 | 必跑时机 | 说明 |
|---|---|---|
| Config | 每次修改 CLI 参数或配置结构 | 防止参数兼容性破坏 |
| PortSet | 每次修改端口解析 | 防止过滤条件错误 |
| Parser | 每次修改 RawRecord/ParsedRecord | 防止 banner 或服务名解析退化 |
| Asset | 每次修改聚合逻辑 | 防止多服务合并错误 |
| Filter | 每次修改过滤规则 | 防止 CIDR/端口规则偏离 MVP |
| Output | 每次修改输出格式 | 防止验收示例不匹配 |
| CLI mock | 每次提交前 | 验证完整主链路 |
| Real mDNS | 发布前或网络模块修改后 | 验证真实网络可用性 |

## 16. 验收通过标准

MVP 测试通过需满足：

- 所有单元测试通过。
- mock text 主链路输出包含示例要求的服务和深度 banner 字段。
- mock json 主链路输出为合法 JSON，且 qdiscover banner 字段完整。
- CIDR 过滤、端口过滤、device-info 保留规则符合文档。
- 非法参数能给出明确错误，且退出码非 0。
- 无真实 mDNS 响应时程序能按 timeout 返回，不阻塞。
- 真实 mDNS 环境下能输出至少基础字段：IP、Port、Hostname、Service、Protocol、TXT banner。
